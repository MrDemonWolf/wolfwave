## Song Requests
**TL;DR: Chat picks the music.**

Everyone can add tracks to your queue. Mods get extra controls.

| Command                  | Who          | What it does                                      |
| ------------------------ | ------------ | ------------------------------------------------- |
| `!songrequest` / `!request <query>` | Everyone     | Request a track by name, or by an Apple Music, Spotify, or YouTube link. `!sr` is the recommended short alias |
| `!sr <query>`            | Everyone     | Recommended short alias for `!songrequest`        |
| `!queue` / `!songlist` / `!requests` | Everyone     | Show how many tracks are queued                   |
| `!myqueue` / `!mysongs`  | Everyone     | Show your own pending requests                    |
| `!skip` / `!next`        | Mods + owner | Skip the current song                             |
| `!hold`                  | Mods + owner | Pause new requests                                |
| `!resume` / `!unhold`    | Mods + owner | Resume a held queue                               |
| `!clearqueue` / `!cq`    | Mods + owner | Wipe the entire queue                             |
| `!playlist`              | Everyone     | Link your song-request playlist (off by default)  |

<Callout type="info">
  Configure the queue, blocklist, and which commands are enabled in **Settings → Song Requests**.
</Callout>

<Callout type="info">
  **`!playlist` setup.** WolfWave gathers requested songs into a **WolfWave Requests** playlist in Apple Music. macOS can't share a playlist for you, so the guided setup (**Settings → Song Requests → Set Up**) walks you through making it public once (in Music: open the playlist, then **Share → Show on Profile**) and grabbing the link. After that `!playlist` posts the link and keeps it current as new requests come in. If you later delete or un-share the playlist, WolfWave turns `!playlist` off and shows a "set up again" banner so chat never gets a dead link. Use **Manage** next to the command, or the banner, to jump back to that step.
</Callout>

## Vote-Skip
**TL;DR: Chat votes a song off.**

Your community skips a track together. No mod needed.

| Command       | Who      | What it does                          |
| ------------- | -------- | ------------------------------------- |
| `!voteskip`   | Everyone | Cast a vote to skip the current song  |
| `!vs`         | Everyone | Short alias for `!voteskip`           |

Enough votes within the time window and the song is skipped. That's the queued
request if one is playing, otherwise the current Apple Music track. If the
window runs out first, the song stays.

Two ways to run a vote, set in **Settings → Song Requests → Chat Vote-Skip**:

- **Chat tally.** `!voteskip` votes are counted until the minimum is reached.
- **Twitch Polls.** A mod's `!voteskip` opens a native Twitch poll (Affiliate/Partner only).

<Callout type="info">
  Set the minimum votes, vote window, cooldown, and subscriber-only voting in
  **Settings → Song Requests → Chat Vote-Skip**.
</Callout>

## Live Stats
**TL;DR: Your listening stats, on demand.**

Viewers ask for your listening stats. Needs the opt-in Listening History on (Settings → History & Stats), where you also pick the window and which facts to show.

| Command   | Who      | What it does                                                                                  |
| --------- | -------- | --------------------------------------------------------------------------------------------- |
| `!stats` / `!musicstats` | Everyone | Replies with your listening stats. Choose the window and which facts to include in Settings   |

The window can be **Today**, **This stream** (since you went live), **Last 7 days**, or **All time**. Pick any mix of play count, listening time, top track, and top artist; a live preview in Settings shows the exact reply.

<Callout type="info">
  Live-gated: replies only fire while your stream is online. WolfWave subscribes to `stream.online` / `stream.offline` EventSub events to track live state, so the command stays quiet when you're offline.
</Callout>

## About WolfWave
**TL;DR: Tell chat what's powering the music.**

A one-tap shoutout for the app. Off by default. Turn it on, then pick the reply that fits your stream.

| Command     | Who      | What it does                                      |
| ----------- | -------- | ------------------------------------------------- |
| `!wolfwave` | Everyone | Replies with what WolfWave is and where to get it |

Four ready-made replies to choose from, each crediting MrDemonWolf and linking to the site:

- **Credit + maker.** Names the app, the maker, and what it does.
- **Viewer how-to.** Points viewers at `!song`, `!last`, and `!sr`.
- **Open-source pitch.** Leads with free and open source for other streamers.
- **Short & friendly.** A one-liner plus the link.

<Callout type="info">
  Enable it and pick a reply in **Settings → Twitch → Bot Commands → !wolfwave Command**. Set cooldowns and add your own aliases there too.
</Callout>

## Channel Points &amp; Bits
**TL;DR: Reward requests without a chat command.**

Two extra ways for viewers to add a track. Both feed the same queue, blocklist, and audience rules as `!sr`.

- **Channel Points reward.** WolfWave creates and manages a "Request a Song" reward on your channel. Redeeming it acts like a `!sr` from that viewer. Fulfilment and cancellation are kept in sync for you.
- **Bit Boost.** When a viewer cheers bits, their most-recent queued request jumps to the front.
- **Recreate Reward.** Deleted the reward by hand on Twitch? Click **Recreate Reward** in the Channel Points & Bits card to make a fresh one. The reward ID shows below the cost picker (masked under Streamer Mode).

<Callout type="info">
  Enable from **Settings → Song Requests → Channel Points & Bits**. Requires Twitch Affiliate or Partner to create the reward.
</Callout>

## Custom Commands
**TL;DR: Make your own commands with a fixed reply.**

Build your own chat commands in **Settings → Twitch → Custom Commands**. Give each one a trigger, a reply, and pick who can use it. Aliases and cooldowns work just like the built-ins.

**Variables** fill in when the command runs:

| Variable | Fills in with |
| --- | --- |
| `$user`, `$sender` | The name of whoever ran the command |
| `$touser` | The first word they typed (a leading `@` is stripped), or their own name if they typed nothing |
| `$args` | Everything they typed after the command |
| `$1`–`$9` | A single word by position |
| `$song` | Your current track |
| `$lastsong` | The track you played before it |

**Example.** Trigger `!hug`, reply `$user gives $touser a big hug!`. A viewer types `!hug @Luna` and chat sees `MrDemonWolf gives Luna a big hug!`.

**Who can use it.** Set each command to Everyone, Subscribers, VIPs, Moderators, or Broadcaster only. More privileged viewers always clear a lower bar (a mod passes a Subscribers-only command).

<Callout type="info">
  Custom commands are saved with your [settings backup](/docs/backup), so they travel with the rest of your setup.
</Callout>

## Custom Aliases
**TL;DR: Type your own command names.**

Supported commands take comma-separated aliases. Add `np, track` to `!song` and viewers can use `!np` or `!track` too.

| Command     | Where to set                                      |
| ----------- | ------------------------------------------------- |
| `!song`     | Settings → Twitch → Bot Commands                  |
| `!last`     | Settings → Twitch → Bot Commands                  |
| `!wolfwave` | Settings → Twitch → Bot Commands                  |
| `!stats`    | Settings → History & Stats → `!stats` Twitch command |
| `!sr`, `!queue`, `!myqueue`, `!skip`, `!clearqueue`, `!voteskip`, `!playlist` | Settings → Song Requests |

Aliases honor the same enable, cooldown, audience, and live-gate rules as the canonical command. The `!` prefix is added for you.

## Cooldowns &amp; Spam
**TL;DR: No spamming allowed.**

Set **Global** and **Per-User** cooldowns in Settings. Keeps chat clean.

- **Global Cooldown** (default 15s): How long everyone waits between commands.
- **User Cooldown** (default 15s): How long one person waits after using a command.

<Callout type="info">
  **Mod Bypass**: Mods and the channel owner always bypass cooldowns.
</Callout>
