The short answers live here. Anything more in-depth links out to the full docs.

## General

<Accordions type="single">

<Accordion title="Does WolfWave work with Spotify?">

No. WolfWave reads Apple Music via `ScriptingBridge`. The same framework Apple uses internally for its own integrations. There is no Spotify hook, and we don't plan to add one. If your streaming setup is Spotify-first, SpotchBot and other Spotify-only tools are a better fit.

</Accordion>

<Accordion title="Is it really free?">

Yes. GPL-3.0 licensed, no paywall, no premium tier, no upsell screens. The full source lives on [GitHub](https://github.com/MrDemonWolf/WolfWave). If you'd like to give back, [sponsor the project](/docs/support).

</Accordion>

<Accordion title="Does it work on Intel Macs?">

No. WolfWave requires Apple Silicon (M1 or later) running macOS 26 (Tahoe) or newer. Intel Macs are not supported.

</Accordion>

<Accordion title="What does the menu bar app actually do?">

It watches Apple Music for the current track, then pushes that data into three places at once:

- **Twitch chat**. Replies to `!song`, `!last`, and the rest.
- **Discord**. A "Listening to WolfWave" Rich Presence card with Apple Music album art and progress.
- **Local WebSocket**. Every play/pause/skip event, ready for an OBS browser source or a custom overlay.

See [Features](/docs/features) for the full list.

</Accordion>

<Accordion title="My !playlist link stopped working, or WolfWave says my playlist needs setting up again">

`!playlist` shares a public link to your **WolfWave Requests** playlist. If you delete that playlist in Apple Music, or turn its sharing off, the link goes dead. WolfWave checks for this when the Song Requests pane opens: it turns `!playlist` off so chat never gets a broken link, and shows a banner at the top of the pane. Tap **Re-share Playlist** (or **Set Up Again**) to fix it in a step or two. Your `!sr` requests, channel points, and bits keep working the whole time.

</Accordion>

</Accordions>

## Streaming &amp; performance

<Accordions type="single">

<Accordion title="Does it slow down my stream?">

No. The app is native Swift, under 30 MB on disk, and event-driven. It reacts to Apple Music's own notifications and falls back to a 2-second poll only if those notifications are missed. CPU and memory impact during a stream are negligible.

</Accordion>

<Accordion title="Will my viewers see ads?">

Never. WolfWave is free and open source. No ads, no upsells, no tracking pixels.

</Accordion>

<Accordion title="Can I use it on a second-PC OBS setup?">

Yes. The local WebSocket binds to all interfaces, so an OBS machine on the same LAN can connect to your Mac as a browser source. Every connection. Loopback or LAN. Must present a per-install token stored in macOS Keychain. See [Security](/docs/security) for the protocol details.

</Accordion>

<Accordion title="Does it work with VTuber overlays / NerdOrDie templates / my own HTML?">

If your overlay can consume a WebSocket feed or a small HTTP endpoint, yes. The data is plain JSON: title, artist, album, artwork URL, position, duration. See [Architecture](/docs/architecture) for the schema.

</Accordion>

<Accordion title="What is Streamer Mode?">

A one-tap toggle (in the menu bar's **Streamer Quick Actions**) that masks sensitive values across WolfWave's UI so the app is safe to show on camera. It hides your connected Twitch channel name, overlay/widget URLs, the WebSocket URI, and the auth token, and disables Copy/Open buttons next to those values. It's UI-only. WebSocket broadcast, Discord Rich Presence, and Twitch chat output keep working.

</Accordion>

</Accordions>

## Security &amp; privacy

<Accordions type="single">

<Accordion title="Is my Twitch token safe?">

Yes. WolfWave uses Twitch's OAuth Device Code flow (no password ever enters the app), stores tokens in macOS Keychain (same place Safari and Mail keep your passwords), and never writes them to disk in plaintext. The app is sandboxed and notarized by Apple.

</Accordion>

<Accordion title="Does WolfWave send any of my data to a server?">

No. There is no WolfWave server. Outbound traffic goes only to services you turn on (Twitch, Discord IPC, iTunes artwork). There is no telemetry or analytics. WolfWave does include an optional, off-by-default diagnostics feature (Settings, Advanced) that uses Apple's MetricKit to capture crash and hang reports locally on your Mac. Those reports are never uploaded to WolfWave servers. See the Privacy Policy for details.

</Accordion>

<Accordion title="How does Listening History store my data?">

Listening History is **off by default**. When you turn it on, WolfWave appends a row to a local NDJSON file inside the app's sandbox for each track that plays past its halfway mark (or 4 minutes, whichever is shorter). Nothing is uploaded. The Stats charts, Monthly Wrap, and `!stats` command all read from that local file. Clear it from **Settings → History & Stats** and the log is gone. See [Security](/docs/security#listening-history-opt-in-on-device) for the full breakdown.

</Accordion>

<Accordion title="Does the Now-Playing Widget URL leak if I screenshare it?">

The widget URL embeds a per-install auth token. If you accidentally show it on stream you can regenerate it from **Settings → Stream Widgets**. That drops every active connection until OBS reconnects with the new URL. Before going live, flip on **Streamer Mode** (see above) to mask the URL in the WolfWave UI entirely.

</Accordion>

<Accordion title="What about App Sandbox?">

WolfWave runs under the macOS App Sandbox with the minimum entitlements needed to control Apple Music via AppleEvents and talk to the Discord IPC socket. See the [`WolfWave.entitlements`](https://github.com/MrDemonWolf/WolfWave/blob/main/apps/native/WolfWave/WolfWave.entitlements) file in the repo for the exact set.

</Accordion>

</Accordions>

## Updates &amp; install

<Accordions type="single">

<Accordion title="How do updates work?">

Sparkle with EdDSA-signed appcasts. The same framework Things, Tower, and many other Mac apps use. The signing key is rotated independently of the developer certificate, so even a compromised release server can't ship an unsigned update.

Homebrew installs are managed by Homebrew instead , `brew upgrade --cask wolfwave` brings you to the latest version.

</Accordion>

<Accordion title="Why isn't WolfWave on the Mac App Store?">

The App Store sandbox blocks the Discord IPC Unix socket, which would break Rich Presence entirely. WolfWave ships outside the store with full Developer ID signing and Apple notarization.

</Accordion>
