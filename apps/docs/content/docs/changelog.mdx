All notable changes to WolfWave are documented here. When an update is ready, the same notes show up right inside the app's update window.


## v2.0.0. Unreleased

### Security
- **Stream Widgets now need an access token.** Your overlay link carries a private token, and the widget server turns away any connection that doesn't have it, so nobody else on your network can pull up your now-playing feed. The token is created on first launch and kept in the macOS Keychain. Existing OBS browser sources keep working with no changes, because WolfWave adds the token to the URL and the served `widget.html` for you. Settings → Stream Widgets lets you reveal it (hidden by default), regenerate it, or paste in your own. Regenerating or editing the token disconnects every widget until it reconnects with the new one.
- **Streamer Mode.** A new tray-menu toggle hides anything you would not want on camera: your connected Twitch channel name, the overlay and widget URLs, the WebSocket address, and the access token. The Copy and Open buttons next to hidden values are switched off, so a screenshot can't leak them either. It only changes what's on screen. Your overlay feed, Discord Rich Presence, and Twitch chat output are untouched.

### Added
- **Back up and restore your settings.** Export your preferences to a plain JSON file from Settings → Advanced, then bring them back on another Mac or after a reinstall. Accounts and secrets are never in the file; on import you decide whether to reconnect Twitch, and saying no skips just that account while the rest of your settings come back. See [Back Up & Restore](/docs/backup).
- **Light, Dark, or System appearance.** Pick one in Settings → General. System follows macOS automatically; Light and Dark override it for the whole app, menu bar menu included. The picker shows System Settings-style preview thumbnails. Settings cards now sit on a solid macOS surface, so they no longer glow too bright in dark mode or wash out in light mode.
- **Discord idle and pause controls.** A new "When not playing" section in Settings → Discord. _Show idle status_ keeps a "Listening to WolfWave · Idle" marker on your profile when nothing's playing instead of clearing it; _Hide track while paused_ clears your profile when you pause. Both are off by default. The Discord preview card now matches real behavior, showing the track only while playing or paused and switching to a matching empty or idle state when playback stops, Apple Music is closed, or Discord isn't running.
- **Chat Vote-Skip.** Viewers can vote to skip the current song with `!voteskip` or `!vs`. Chat-tally mode counts unique voters against a minimum threshold; opt-in Twitch Polls mode opens a native poll (Affiliate/Partner). Configure it in Settings → Song Requests → Chat Vote-Skip.
- **Listening History.** An opt-in, on-device log of the tracks you play. It stays on your Mac. Off by default; turn it on in Settings or onboarding.
- **Stats and Charts.** Top artists, listening time, a 7-day trend, and a listening-by-hour breakdown in the new History & Stats section. Laid out as a two-column dashboard that drops to one column on a narrow window, with the summary tiles beside a today's-top-track highlight and one Top card that switches between artists, tracks, and albums.
- **Monthly Wrap.** A personal "wrapped"-style summary for any month. Save it as a PNG or send it through the macOS share sheet (Messages, Mail, AirDrop).
- **`!stats` command.** Viewers can ask for today's top track in chat. It only replies while your stream is live.
- **Song Requests.** Viewers can request songs with `!sr <song>` in Twitch chat. Requests play through Apple Music without stealing focus from your stream.
- **`!queue` / `!myqueue`.** Show the whole queue, or a viewer's own requests, in chat.
- **`!skip` / `!next`.** Mods and the broadcaster can skip the current request.
- **`!clearqueue`.** Mods and the broadcaster can wipe the queue (with a confirmation first).
- **`!hold` / `!resume`.** Mods and the broadcaster can pause the queue; requests pile up without auto-playing so you can curate before releasing them.
- **Hold controls in the app.** A Hold/Resume button in the Queue view and a matching menu bar toggle.
- **Requests wait while Apple Music is closed.** They're saved when Music is closed and play once it reopens.
- **Fallback playlist.** Pick an Apple Music playlist to play whenever the request queue runs dry.
- **Song Request Queue view.** A now-playing card, position badges, who requested each track, and Skip / Hold / Clear controls.
- **Per-user limits, subscriber-only mode, command aliases**, and on/off toggles per command.
- **Custom aliases for `!song`, `!last`, and `!stats`.** Comma-separated alias fields in Settings → Twitch and Settings → History & Stats. Aliases follow the same cooldown and live-only rules as the original command.
- **Hold-queue toggle in Song Request settings.** Flip the queue between Hold and Playing right from the Playback card. Before this it was only reachable from the menu bar.
- **Recreate Reward button.** Clears the saved Channel-Point reward and makes WolfWave build a fresh "Request a Song" reward on Twitch. Use it if you delete the reward by hand. The reward ID shows below the cost picker (hidden when Streamer Mode is on).
- **History retention picker.** Keep your listening history for Forever / 7 / 30 / 90 / 180 / 365 days. Older entries are cleared at the next launch.
- **macOS 26 Liquid Glass onboarding.** A full rebuild of the first-launch wizard with two new steps (Menu Bar Pointer and OBS Widget) and Liquid Glass materials to match Tahoe's look.
- **Apple Music onboarding step.** The wizard asks for Apple Music library access for song search, with a clear path back if you decline.
- **OBS Widget onboarding step.** Preview the overlay URL and flip on the HTTP widget during setup, so you can grab your overlay link before you finish.
- **Faster overlay URLs.** WolfWave caches your local IP so settings views no longer pause while looking it up.
- **Diagnostics and bug reporting.** Export Logs, Clear Logs, and a one-click Report a Bug button in Advanced settings; the issue comes pre-filled with sensitive details already redacted.
- **Branded About window.** A native WolfWave About window replaces the default macOS panel.
- **Apple Music logo on the Music access row.** Once access is granted, the row shows the Apple Music mark.
- **Widget themes.** Six overlay-widget themes (Default, Dark, Light, Glass, Neon, WolfWave) and three layouts (Horizontal, Vertical, Compact).
- **Discord presence polish.** Friendlier presence buttons, a live preview in Discord settings, and cleaner connection handling.
- **Song-change notifications.** An opt-in macOS notification on every track change, with the track name, artist, and album art. A new banner replaces the last one instead of stacking. Off by default in Settings → General → Notifications.
- **Skip-vote notifications.** Opt-in macOS notifications when a chat skip-vote starts (silent, shows the target or the open poll) and when it passes (names the skipped track, default sound). Off by default in Settings → Notifications; they stay off until Chat Vote-Skip is on. The onboarding Permissions step also lets you pick which alerts you want.
- **On-device diagnostics.** An opt-in MetricKit report in Advanced settings with a share card for attaching one to a bug report. Reports stay on your Mac.
- **Discord playlist in your presence.** Discord Rich Presence now shows the current Apple Music playlist name next to the track.
- **WolfMark branding.** A new placeholder mark, a branded download page, and refreshed repo README and SECURITY docs.
- **Smoother OBS widget.** The overlay slides in with a small bounce when a song starts and slides and blurs out calmly when playback stops, with the progress bar draining to zero. Fast track skips crossfade the artwork and text instead of replaying the entrance, so there's no strobing on stream. Pausing keeps the card up.

### Changed
- **Calmer destructive buttons.** Reset, Clear Logs, and Clear Artwork Cache drop the loud red bar for a neutral button with a red label, and the Danger Zone card trades its red wash for a clean card with a red heading. Red stays on the label, heading, and final confirm.
- **Type RESET to wipe everything.** The Reset All Settings confirmation spells out what it erases and makes you type RESET before the button unlocks, so a stray click can't reset the app.
- **Redesigned Settings sidebar.** A VoiceInk-style sidebar with a branded header (mark, name, version), grouped rows with round icon chips, and a solid accent-colored selection pill with a hover highlight. Grouping, keyboard navigation, and VoiceOver are unchanged.
- **Apple Music playback keeps your focus.** Music never steals focus from OBS or other streaming tools.
- **MusicKit is only used for search**, never for playback.
- **Settings split into per-section files** for snappier redraws and easier editing.
- **Onboarding polish.** Repeated success rows removed, Apple Music permission copy tightened, button and step sizes pinned so nothing jumps around.
- **OBS onboarding step** collapses to a single toggle, with the overlay URL promoted to the top.
- **Tighter wording** across onboarding, settings, and the menu bar.
- **Notifications toggle** switches itself off once permission is granted; the denied state reads the same across the wizard.
- **Permissions live in one onboarding step.** Apple Music access, the notification prompt, and the per-alert toggles are now in a single "Permissions" step.
- **`PillButton`** now uses the macOS-standard rounded rectangle instead of a `Capsule`.
- **Menu bar preview icon** renders crisply at every screen density.
- **Settings window** sized to fit cleanly on 720p and 1080p displays.
- **Music permission row is actionable.** Tap it to re-request access or jump into System Settings.
- **Test suite tidied up and grown** to more than 880 tests across 86 files.
- **Design tokens everywhere.** Settings, onboarding, and shared views now read sizes and spacing from the generated design tokens instead of hardcoded numbers, for consistent typography and spacing across every tab.
- **Concurrency cleanup.** Leftover `DispatchQueue.main.async` calls in menu-bar, song-request, and Discord settings now use structured `Task { @MainActor }` for Swift 6 cleanliness.
- **Logger redaction literals** moved to compile-checked `#/.../#` patterns; no more `try! Regex(...)` warnings.
- **Liquid Glass onboarding surface.** The wizard panel now renders on a `.glassEffect(.regular)` material, matching the settings cards and now-playing card.
- **Streamer Mode hides your Twitch account name** in Settings too, matching the channel-ID rows and the menu bar.
- **Onboarding accessibility.** VoiceOver now announces each step by name ("Step 3 of 7: Twitch") via the progress dots.
- **Sidebar grouped by purpose.** Sections are grouped by what they're for, and the App group is ordered by how often you reach for it. App Visibility is now a single-column picker for the menu bar, the Dock, or both, and the General tab lays Startup and Display Mode side by side.

### Performance
- **Lower idle energy use.** WolfWave's periodic timers (fallback poll, WebSocket progress tick, Discord poll, song-request auto-advance) now let macOS batch their wakeups together, which saves power for an all-day menu bar app. Real-time track changes still arrive instantly; only the background polling changed.
- **No more jank** when switching between settings sections.
- **Faster first paint.** Font lookup moved off the main thread so the Widget Setup row no longer blocks it.
- **Instant Now-Playing Server row** thanks to the cached local IP lookup.

### Developer

_Developer-facing changes. Not visible to end users._

- **Unified design system.** A single `design-system/tokens.json` feeds a generator that emits four platform outputs (Swift, docs CSS, widget JS, marketing TS); a Turbo `tokens` task runs as a build prerequisite.
- **Component catalog.** One markdown entry per reusable view under `design-system/components/`.
- **DEBUG-only Debug settings tab.** Developer tooling (inspectors, service controls, log/event views, UI previews) plus What's New preview controls, only in DEBUG builds.
- **Expanded design-token roster.** `font.size` now exposes 9/15/16/18/24/26/28/36 px slots and `space` adds `s0` (2 px), `s10` (32 px), and `s11` (44 px). Token-only additions with no visual change.
- **Sparkle delegate hardening.** Opted out of Sparkle's system-profile telemetry (`allowedSystemProfileKeys` returns empty); explicit consent is documented as the contract for `automaticallyDownloadsUpdates = false`.
- **Sparkle release notes.** The release pipeline turns each version's CHANGELOG section into styled HTML (`scripts/release-notes.mjs`) and embeds it in the appcast, so the in-app update dialog shows what's new with a link out to this changelog.
- **WebSocket security-model doc.** Added a `## Security model` block to `WebSocketServerService.swift` describing the loopback-only contract.
- **New unit tests.** `SongBlocklistTests`, `HistoryFormattingTests`, and `LaunchAtLoginServiceTests` cover three previously-untested services.
- **OBS widget build pipeline.** The widget source moved into a new `apps/widget/` Tailwind + TypeScript workspace, and the bundled `widget.html` is now a generated artifact with the compiled CSS, design tokens, and JS runtime all inlined into one self-contained file. Xcode auto-rebuilds it via a pre-build Run Script phase; CI rebuilds it before every `xcodebuild` run. Manual command: `bun run --filter widget build`. The WebSocket payload, widget URL, and theme/layout params are unchanged. See [OBS Widget Architecture](/docs/widget) for the pipeline and transition state machine.
- **Inside-out release signing.** `build_release.yml` signs nested bundles deepest-first before the app shell, dropping `--deep --force`, which stripped per-bundle entitlements.
- **Typed NotificationCenter payload helpers.** `Core/NotificationPayloads.swift` centralizes every notification payload post/observe pair so both sides share the same keys and types.
- **HelixClient.** A shared HTTP wrapper for all Helix API calls, replacing per-service `send()` helpers. Maps 401, 429, and structured Helix errors uniformly.
- **Preferences typed accessor.** `Core/Preferences.swift` centralizes UserDefaults reads and writes for non-bool keys across AppDelegate and WolfWaveApp.
- **Actor adoption.** `SongBlocklist` and `SkipVoteManager` move from `final class + NSLock` to actors; `BotCommandContext` is marked `nonisolated + Sendable`. Combine is fully removed from the app target.

### Fixed
- **One recovery card for denied Music access.** Settings → General no longer repeats the same "Music access denied" warning in three places. A single card owns the explanation and the Open System Settings button, the Sync Music card hides its duplicate row while denied, and the now-playing card shows a calm orange "Paused" instead of a red "Denied". "Try again" shows a spinner and a "still off" hint instead of feeling dead (#240).
- **Twitch bot no longer goes silent after a relaunch.** It now reconnects to your saved channel on its own once the token check passes, so chat commands work without reopening the Twitch settings pane. The reconnect is silent, so no "online" chat ping fires unless you click Connect yourself.
- **Keychain self-heals a duplicate item.** `upsertItem` recovers from `errSecDuplicateItem` (OSStatus -25299) when an older entry uses a mismatched accessibility class, fixing the Stream Widgets token write after onboarding.
- **Twitch settings card stops jumping.** It no longer resizes between the "Not signed in", "Authorizing", and "Error" states. A reserved height plus symmetric transitions; the unused `TwitchReauthView` was deleted so re-auth shows up inline in the same card.
- **Onboarding steps are centered again.** Step content sat pinned to the top with empty space below; the scroll view is now constrained to the window so the centering layout works. The reserved icon slot is center-aligned too, so compact steps (the Discord and Preferences toggles) no longer leave dead space below them.
- **Two-PC stream widget works.** The overlay WebSocket and widget HTTP server now listen on every network interface (not just loopback), and `widget.html` connects to its own hostname instead of a hardcoded `localhost`. Open `http://<your-lan-ip>:7780` from a second computer or your phone on the same network and the widget loads (gated by the new access token so LAN exposure stays safe).
- **GitHub URL config plumbing.** `GITHUB_REPO_OWNER` and `GITHUB_REPO_NAME` from `Config.xcconfig` now flow through `Info.plist`, so Report a Bug and releases links come from your config.
- **Intel Homebrew detection.** Intel Macs installed via Homebrew now disable Sparkle and show the Homebrew update card; the old check missed the `/usr/local/Caskroom/` location casks use.
- Cleaned up native build warnings, a SwiftUI layout reentrancy issue, and duplicate log lines.
- Fixed an onboarding URL bug, permissions correctness, and layout stability.
- Export Logs no longer crashes on empty or very large logs.
- Settings and About windows show after the AppKit layout pass, silencing the recursion warning.
- Music access icon adapts to light and dark mode; the top scroll fade is gone.
- PillButton no longer shifts width mid-animation.
- Apple Music and OBS logos trimmed to their glyphs so template tinting works.
- Menu bar pointer arrow anchored to the TrayIcon center in onboarding.
- Brand icons in the menu bar render as templates with refreshed logo SVGs.
- Settings sidebar toggle. Removed the duplicate titlebar toggle, restored the toolbar toggle, and got rid of the floating `>>` reveal control.
- About window spacer gap removed; app icon cutouts preserved via the `evenodd` fill-rule.
- Apple Music control under the sandbox. Granted the Apple Events entitlement so the permission prompt actually appears and ScriptingBridge can drive Music.
- Now-playing survives ScriptingBridge quirks. The player-state parser accepts every value type Music returns instead of silently blanking the card, Discord presence, and overlay.
- Recheck and the Sync toggle force a fresh now-playing read so the UI catches up right away.
- Apple Music Recheck button visibly responds when tapped.
- Stats card layout no longer jumps around, and the section is gated on Apple Music access.
- Settings window default size enlarged; `StatusChip` tightened so integration rows fit without horizontal scrolling.
- About window Check for Updates now actually invokes Sparkle; the Release Notes link points at this changelog.
- Onboarding Preferences step no longer overlaps the nav bar on shorter windows.
- Onboarding step header anchored so icons stop drifting between steps.
- **Vote-skip timer race.** A timer that outlived a fast-passing vote could clear a newer session when the cooldown is 0. It now captures a session token when it starts and bails if a different session has opened since.
