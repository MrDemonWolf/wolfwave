## Review findings

Four review passes ran: Swift + Swift 6 concurrency, SwiftUI + macOS HIG + design tokens, duplication, and tests. Each checked the code against current Apple docs (Swift API Design Guidelines, Swift 6 strict concurrency, macOS HIG, SwiftUI Settings + accessibility, macOS 26 Liquid Glass).

### Verdict: healthy

No P0 blockers in any pass. The codebase is well factored. Specifically confirmed:

- **Opaque cards rule holds.** Zero `.glassEffect` anywhere. `CardModifier` uses `controlBackgroundColor` + hairline. The only materials are 3 `.regularMaterial` uses in onboarding chrome, which is HIG correct. (This is why the changelog was wrong and is now fixed.)
- **Concurrency claims hold.** `SongBlocklist` and `SkipVoteManager` are real actors. `BotCommandContext` is `nonisolated + Sendable`. Combine is fully removed. No `try!`, no force-casts.
- **The locked-in `playerState` parser and entitlements are intact.**

### Punch-list (applied and verified)

Once the real `Config.xcconfig` was supplied, three coordinated agents applied this list on disjoint file sets, and a full `make test` run verified it: 2,070 tests, 0 failures. The items below are the record of what changed. A short set of intentionally deferred items follows at the end.

**Swift / concurrency**

- P1 `Services/SongRequest/SongRequestService.swift:197-243` reentrancy gap: two near-simultaneous `!sr` while Music is closed can both call `playNextInQueue()`. Guard with an `isStartingPlayback` flag set without an intervening `await`.
- P2 `Services/Discord/DiscordRPCService.swift:938-1068` does blocking `Darwin.read/write` on the actor executor. Move raw socket I/O off the actor.
- P2 redundant `NSLock` on types that are already MainActor-isolated (`SongRequestQueue`, `CooldownManager`, `BotCommandDispatcher`, `TrackInfoCommand`). At minimum fix the misleading "safe from any thread" doc comments.
- P2 mark pure value types `Sendable`: `SongRequestItem`, `RecentTrack`, `TwitchDeviceAuth`, `TwitchDeviceCodeResponse`.
- Helper: add `Preferences.int(_:default:)` / `Preferences.bool(_:default:)` and route ~15 ad-hoc default-value reads through them.

**SwiftUI / HIG / design**

- P1 token lint is under-enforced. `spacing: 6` appears 45x with no token (DS jumps 4 to 8): add `DSSpace.s1h = 6` or formally bless 6. `WebSocket/WebSocketSettingsView.swift:442 .padding(.vertical, 12)` to `DSSpace.s4`. The lint also exempts the whole `Onboarding/` tree (~19 tokenizable literals): decide if onboarding is a separate language and document it, or drop the carve-out.
- P1 `Shared/ToggleSettingRow.swift:52` uses `.toggleStyle(.switch)`. macOS HIG prefers checkboxes for settings. Defensible house style (Tahoe System Settings uses switches too), but the app is split: `AppVisibilitySettingsView` uses `.checkbox`. Pick one. Suggest a `style:` param defaulting to `.checkbox`.
- P1 zero Dynamic Type support app-wide (all fonts fixed-point via tokens). Track as an accessibility initiative.
- P2 `Shared/IntegrationDashboardView.swift` type-erases icons with `AnyView` 5x. Make `row` generic over `Icon: View`.
- P2 ~20 `RoundedRectangle(cornerRadius: 12)` card sites are off-token: add `DSRadius.lg2 = 12`. Icon `.frame(width: 16/22/36)` literals: add a `DSDimension.Icon` scale.
- P2 `Shared/ConfigRequiredBanner.swift` duplicates `CalloutBanner` (fold it in). Two onboarding eyebrows hand-roll the deprecated ALL-CAPS + `.tracking` pattern: use `.sectionEyebrow()`.
- P2 empty-string accessibility attrs in `DSIconButton`, `CopyButton`, `ToggleSettingRow`: apply the modifier only when the value is non-nil.

**Duplication (remaining clusters, ranked)**

- HIGH `Views/WebSocket/WebSocketSettingsView.swift` hand-rolls the masked-URL + Copy row 4x (253-274, 280-309, 642-679, 685-729). Extract `CopyableURLRow`. ~80 lines.
- HIGH finish the `ExternalLink.open` migration: ~13 remaining `URL(string:) + NSWorkspace.open` sites (About, DeviceCodeView, UpdateBannerView, OnboardingNotifications, MusicPermissionState, NotificationService, AppDelegate+MenuBar).
- MED `Services/Twitch/TwitchChatService.swift` reimplements `helixHeaders` + inline header triples 5x. Route through `HelixClient.headers(for:)`.
- MED `TwitchChatService.subscribeToChannelChatMessage` (1600-1642) rebuilds the EventSub POST. Give `postEventSubSubscription` an `onSuccess:` closure and call it.
- MED `Services/WebSocket/WidgetHTTPService.swift` builds the raw HTTP response 4x. Extract `sendHTTP(to:status:contentType:body:extraHeaders:)`.
- LOW `Views/Twitch/DeviceCodeView.swift` uses leak-prone `NSCursor.push/pop` (use `.pointerCursor()`) and a hand-rolled copy button (use `CopyButton`).

**Tests**

- Trim `AppConstantsTests` + `AppConstantsEdgeCaseTests` + `AppConstantsConfigOverrideTests` (45) down to the invariant checks (URL parseability, min <= ideal, key registration). Drop the literal-value restatements.
- Merge `WebSocketAuthTokenTests` into `WebSocketServerAuthTests` (keep only the unique `generate()` coverage).
- Parameterize `KeychainServiceTests` token variants with the existing `assertKeychainRoundTrip` helper.
- Fix fixed-sleep flakiness: `SongRequestServiceTests.swift:384` (2.5s), `SkipVoteManagerTests.swift:210` (2s), `WidgetHTTPServiceTests.swift:124/147`, `WebSocketServerIntegrationTests` (fixed ports 59001-59010). Inject the interval/clock or poll-until a signal.
- GAPS worth adding: notification dedup (assert the built request reuses the per-type identifier), `handleVoteEvent` gating (extract a pure predicate and test it), monthly-wrap edge cases (empty month, single play, month boundary).
- Migrate the 12 files that hand-roll `UserDefaults.removeObject` to `resetAllSettings()` in `tearDown`.

### Intentionally deferred (your call, not missed)

- **ToggleSettingRow `.switch` to `.checkbox`**. HIG leans checkbox for settings, but Tahoe System Settings uses switches too. House-style decision, so left as-is.
- **Dynamic Type / text scaling**. The whole UI is fixed-point via tokens. A real accessibility initiative, not a one-PR change.
- **DiscordRPCService blocking socket I/O off the actor**. A substantial rewrite of the IPC read/write path. Higher risk than the rest, so deferred.
- **A few `resetAllSettings()` test migrations**. Three files that extend plain `XCTestCase` with their own reset helpers were left so their base class does not change.
- **Extend `ds-lint` to flag literal `6` and `cornerRadius`/`frame`**. The literals are tokenized now, but the linter regex still does not catch them, so future drift could slip through.

### Apple docs grounded against

Swift API Design Guidelines; Swift 6 strict concurrency / data-race safety; macOS HIG (Settings, menu bar extras, materials, typography); SwiftUI `Settings` scene + `SettingsLink`; macOS 26 Tahoe Liquid Glass best practices; SwiftUI accessibility (VoiceOver, Dynamic Type).
